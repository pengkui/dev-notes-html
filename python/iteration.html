• Iterable: A container object (e.g. list, tuple, set, str, dict, file, etc.) that supports iterating over its members (via for statement, zip(), map(), etc.) in a deterministic order. This definition is by behavior; and there is no such type called iterable per se. Operationally, an iterable must define __iter__() or __getitem__() method.
  • Iterator: An iterable object with both next() and __iter__() methods implemented. Unlike non-iterator iterables, an iterator's __iter__() method returns itself. For example, a dict is a non-iterator (and non-sequence BTW) iterable, which does not have a next() method but can be iterated via a for-statement.
  • Generator: A special function that behaves as an iterator (note: the official glossary document is wrong at saying that it "returns an iterator"). "It looks like a normal function except that it contains yield statements for producing a series a values usable in a for-loop or that can be retrieved one at a time with the next() function."
